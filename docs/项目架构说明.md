# 测试平台远程执行任务 - 项目架构说明

## 1. 架构概述

本项目采用**分层架构 + 适配器模式**，实现统一的测试任务调度平台，支持多种测试工具（CANoe、TSMaster、TTworkbench）的自动化测试。

### 1.1 架构特点

- **分层解耦**: 数据层、服务层、执行层清晰分离
- **插件化设计**: 通过适配器模式支持多种测试工具灵活接入
- **统一接口**: Java端与Python端通过标准化WebSocket协议通信
- **可扩展性**: 预留新测试工具接入接口

## 2. 核心模块设计

## 3. 适配器模式详解

### 3.1 适配器基类

```python
class BaseTestAdapter(ABC):
    """测试工具适配器基类"""
    
    @abstractmethod
    def connect(self) -> bool:
        """连接测试工具"""
        pass
    
    @abstractmethod
    def disconnect(self) -> bool:
        """断开测试工具连接"""
        pass
    
    @abstractmethod
    def load_configuration(self, config_path: str) -> bool:
        """加载测试配置文件"""
        pass
    
    @abstractmethod
    def start_test(self) -> bool:
        """启动测试（测量/仿真）"""
        pass
    
    @abstractmethod
    def stop_test(self) -> bool:
        """停止测试（测量/仿真）"""
        pass
    
    @abstractmethod
    def execute_test_item(self, item: Dict[str, Any]) -> Dict[str, Any]:
        """执行单个测试项"""
        pass
```

### 3.2 适配器工厂

```python
# 适配器注册表
ADAPTER_REGISTRY = {
    TestToolType.CANOE: CANoeAdapter,
    TestToolType.TSMASTER: TSMasterAdapter,
    TestToolType.TTWORKBENCH: TTworkbenchAdapter,
}

def create_adapter(tool_type: TestToolType, config: dict = None) -> BaseTestAdapter:
    """创建适配器实例"""
    adapter_class = ADAPTER_REGISTRY[tool_type]
    return adapter_class(config)
```

### 3.3 具体适配器实现

#### CANoe适配器
- **技术**: COM接口 (pywin32)
- **功能**: 测量控制、信号读写、系统变量操作、测试模块执行
- **适用**: CANoe 10/11/12/13/14/15/16/17

#### TSMaster适配器
- **技术**: Python API
- **功能**: 总线仿真、信号读写、报文收发、C脚本调用
- **适用**: TSMaster全版本

#### TTworkbench适配器
- **技术**: TTman命令行调用
- **功能**: clf文件执行、批量测试、报告获取
- **适用**: TTworkbench (TC8/AVB/AUTOSAR测试)

## 4. 数据流向设计

### 4.1 Excel导入流程

```
Excel文件 → 文件上传 → 表头校验 → 字段解析 → 字段分解 → 数据校验 → JSON转换 → API传输 → Python执行
```

### 4.2 任务执行流程

```
Java服务端 → WebSocket下发 → Python接收 → 工具适配器选择 → 测试执行 → 结果采集 → WebSocket上报 → Java处理
```

### 4.3 测试系统机柜接入方案

#### 网络自动化测试系统机柜 (CANoe 10)
- **接入方式**: C#中间层 + Python调用
- **架构**: Java → Python → C#程序 → CANoe 10
- **特点**: 复用原有C#源码，增加API接口层

#### 便携式网络自动化测试系统机柜 (CANoe 17)
- **接入方式**: Python直接调用CANoe COM接口
- **架构**: Java → Python → CANoe COM → CANoe 17
- **特点**: 原生COM接口，稳定性高

#### 车载以太网测试系统机柜 (TTworkbench)
- **接入方式**: Python调用TTman命令行
- **架构**: Java → Python → TTman命令 → TTworkbench
- **特点**: 通过clf文件执行，支持TC8/AVB/AUTOSAR测试

- `platform`：平台名称，固定为"NETWORK"（必填）
- `caseList`：用例结果集合（必填）

  - `result`：结果状态，取值范围：PASS/FAIL/BLOCK/SKIP（必填）
  - `remark`：备注信息（可选）
  - `created`：测试执行时间，格式：YYYY-MM-DD HH:MM:SS（可选）
  - `reportPath`：测试报告路径（可选）

## 7. 关键技术点

### 7.1 适配器模式优势

1. **统一接口**: 所有测试工具通过统一接口调用
2. **易于扩展**: 新增测试工具只需实现适配器接口
3. **解耦设计**: 业务逻辑与具体工具实现解耦
4. **便于维护**: 各适配器独立维护，互不影响

### 7.2 WebSocket长连接

- **实时双向通信**: 支持状态同步和日志推送
- **心跳检测**: 保持连接活跃，检测设备在线状态
- **断线重连**: 自动重连机制，提高可靠性

### 7.3 异步任务执行

- **非阻塞**: 避免阻塞主线程
- **任务取消**: 支持优雅停止正在执行的任务
- **超时控制**: 防止任务无限期执行

## 9. 总结

本项目架构通过**适配器模式**实现了多种测试工具的统一接入，通过**分层架构**实现了系统的解耦和可扩展性。主要优势包括：

1. **灵活性**: 支持CANoe、TSMaster、TTworkbench等多种测试工具
2. **可扩展性**: 易于添加新的测试工具支持
3. **可维护性**: 模块化设计，各组件职责清晰
4. **可靠性**: WebSocket长连接、心跳检测、断线重连等机制保障通信稳定
