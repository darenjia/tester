## 1. 架构设计概述

根据架构文档，系统应该包含以下核心组件：

### 1.1 Python执行器结构（设计）

```
python_executor/
├── app.py                       # Flask应用入口
├── config/
│   └── settings.py              # 配置文件
├── core/                        # 核心模块
│   ├── task_executor.py         # 任务执行引擎
│   ├── result_collector.py      # 结果收集器
│   └── adapters/                # 测试工具适配器
│       ├── __init__.py          # 适配器工厂
│       ├── base_adapter.py      # 适配器基类
│       ├── canoe_adapter.py     # CANoe适配器
│       ├── tsmaster_adapter.py  # TSMaster适配器
│       └── ttworkbench_adapter.py # TTworkbench适配器
├── websocket/
│   └── client.py                # WebSocket客户端
├── models/
│   ├── task.py                  # 任务模型
│   └── result.py                # 结果模型
└── utils/
    ├── logger.py                # 日志工具
    └── validators.py            # 输入验证
```

### 1.2 通信架构（设计）

- **Java服务端** ↔ **Python执行器** (WebSocket长连接)
- **心跳检测**: 30秒间隔
- **消息类型**: HEARTBEAT, TASK_DISPATCH, TASK_STATUS, LOG_STREAM, RESULT_REPORT, TASK_CANCEL
