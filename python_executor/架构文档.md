# 测试执行器系统架构文档

## 1. 系统概述

测试执行器（TestExecutor）是一个用于自动化测试的桌面应用程序，支持多种测试工具（CANoe、TSMaster、TTworkbench）的测试用例执行管理。该系统采用分层架构设计，支持任务调度、配置管理、用例映射、执行监控等功能。

### 1.1 技术栈

| 层级 | 技术/框架 |
|------|-----------|
| 桌面框架 | PyWebView + Flask |
| Web框架 | Flask + Jinja2模板 |
| 后端语言 | Python 3.x |
| 前端 | HTML5 + CSS3 + JavaScript |
| 打包工具 | PyInstaller |

### 4.3 核心业务层 (core/)

#### 4.3.1 任务执行器 (task_executor_production.py)

**职责**: 任务执行的核心引擎，支持生产环境的熔断、重试、性能监控等功能

**核心流程**:
```
1. 接收任务(Task)
2. 从任务中取出测试用例
3. 根据用例编号查询用例映射
4. 获取cfg工程文件路径(script_path)
5. 从映射配置路径复制cfg文件(如不存在)
6. 根据用例配置生成ini文件
7. 通过适配器执行测试
8. 收集结果并上报
```

**核心方法**:
```python
class TaskExecutorProduction:
    def _execute_task_production(task: Task)     # 生产环境任务执行
    def _execute_test_items_with_config(task)   # 配置驱动的用例执行
    def _connect_tool_with_retry(task)         # 重试连接工具
    def _load_configuration_by_path(path)       # 加载配置文件
```

#### 4.3.2 用例映射管理器 (case_mapping_manager.py)

**职责**: 管理接口用例名称与脚本Case编号的映射关系

**核心方法**:
```python
class CaseMappingManager:
    def register_mapping(mapping, changed_by, reason)     # 注册映射
    def get_mapping(case_no)                             # 通过编号查询
    def get_mapping_by_name(name)                         # 通过名称模糊查询
    def update_case(case_no, updates, changed_by, reason) # 更新映射
    def delete_case(case_no, changed_by, reason)          # 删除映射
    def import_mappings(data, overwrite)                  # 批量导入
    def export_mappings()                                 # 导出演射
```

#### 4.3.3 配置管理器 (config_manager.py - TestConfigManager)

**职责**: 管理测试配置文件，生成ini文件

**核心方法**:
```python
class TestConfigManager:
    def prepare_config_for_task(task)                       # 为任务准备配置
    def prepare_config_for_case(case_no, script_path, ini_config)  # 为单个用例准备配置
    def _generate_ini_from_case_config(case_no, ini_config) # 从用例映射生成ini
    def cache_config(config_name, source_path)             # 缓存配置文件
```

#### 4.3.4 任务调度器 (task_scheduler.py)

**职责**: 调度任务的执行，管理任务队列
