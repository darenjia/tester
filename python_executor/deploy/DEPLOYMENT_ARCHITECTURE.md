# Python执行器部署架构方案

## 1. 部署场景分析

### 1.1 环境特点
- **分布式部署**: 多台测试设备，每台运行独立执行器
- **生产环境**: 需要稳定性和基本监控
- **半自动化**: PowerShell脚本一键部署

### 1.2 部署要求
- 每台设备独立部署Python执行器
- 统一配置管理
- 集中日志收集
- 远程监控能力

## 3. 部署模式

### 3.1 模式一：Windows服务部署（推荐）

**适用场景**: 长期运行的生产环境

**优点**:
- 系统启动自动运行
- 崩溃自动重启
- 统一的服务管理

**缺点**:
- 需要管理员权限
- 调试相对复杂

### 3.2 模式二：计划任务部署

**适用场景**: 定时执行的测试任务

**优点**:
- 灵活的时间调度
- 可与用户交互

**缺点**:
- 不适合实时任务
- 需要用户登录

### 3.3 模式三：手动启动

**适用场景**: 开发和调试

**优点**:
- 简单直接
- 便于调试

**缺点**:
- 需要手动维护
- 无法自动恢复

## 4. 配置管理策略

### 4.1 配置分层

```
配置优先级（从高到低）:
1. 环境变量
2. 本地配置文件 (config/local.json)
3. 设备特定配置 (config/device_{hostname}.json)
4. 全局配置 (config/global.json)
5. 默认配置 (代码内置)
```

### 4.2 配置同步方案

**方案A：集中配置服务器**
- 部署配置中心服务
- 执行器启动时拉取配置
- 支持配置热更新

**方案B：配置文件分发**
- 使用Ansible/SCCM分发配置
- 定期同步配置文件
- 版本控制管理

**方案C：混合方案（推荐）**
- 基础配置本地存储
- 动态配置从服务器获取
- 关键配置支持热更新

## 5. 日志和监控方案

### 5.1 日志收集

```
本地日志 → 文件轮转 → 集中收集 → 日志分析
   │           │            │           │
   ▼           ▼            ▼           ▼
日志文件    按天/大小    Filebeat/    ELK Stack
          自动切割     自定义上传    /Splunk
```

### 5.2 监控方案

**轻量级监控（推荐）**:
- 内置性能指标收集
- HTTP端点暴露指标
- 简单的告警机制

**企业级监控（可选）**:
- Prometheus + Grafana
- Zabbix/Nagios
- 自定义监控平台

## 6. 安全考虑

### 6.1 通信安全
- WebSocket使用WSS (TLS加密)
- API认证 (Token/API Key)
- 防火墙规则限制访问

### 6.2 文件安全
- 配置文件权限控制
- 日志文件定期清理
- 敏感信息加密存储

### 6.3 运行安全
- 服务账户权限最小化
- 定期更新依赖库
- 安全审计日志

## 7. 高可用考虑

### 7.1 单节点高可用
- 服务崩溃自动重启
- 连接断开自动重连
- 任务失败自动重试

### 7.2 多节点考虑（未来扩展）
- 任务分配负载均衡
- 节点故障自动切换
- 状态同步机制

## 8. 部署流程

### 8.1 首次部署流程

```
1. 环境准备
   └─ 检查Python版本、安装依赖

2. 配置准备
   └─ 生成设备特定配置

3. 安装服务
   └─ 注册Windows服务/创建计划任务

4. 启动验证
   └─ 检查服务状态、测试连接

5. 监控配置
   └─ 配置日志收集、告警规则
```

### 8.2 升级流程

```
1. 备份当前版本
   └─ 备份代码、配置、日志

2. 下载新版本
   └─ 获取最新代码/安装包

3. 停止服务
   └─ 优雅停止当前服务

4. 更新代码
   └─ 替换代码文件

5. 更新依赖
   └─ 安装新依赖

6. 启动服务
   └─ 启动新版本服务

7. 验证测试
   └─ 功能验证、连接测试
```

## 9. 回滚策略

### 9.1 自动回滚条件
- 服务启动失败
- 连接测试失败
- 健康检查失败

### 9.2 手动回滚步骤
```powershell
# 1. 停止当前服务
Stop-Service PythonExecutor

# 2. 恢复备份
Restore-Deployment -BackupPath "backup_20240115"

# 3. 启动服务
Start-Service PythonExecutor

# 4. 验证
Test-Deployment
```

## 12. 故障处理

### 常见问题

#### 服务无法启动
```powershell
# 检查日志
Get-Content logs/executor.log -Tail 50

# 检查依赖
pip list | grep -E "flask|socketio"

# 检查端口占用
Get-NetTCPConnection -LocalPort 8180
```

#### 连接测试平台失败
```powershell
# 测试网络连通
Test-NetConnection -ComputerName platform-server -Port 8180

# 检查配置
Get-Content config/executor_config.json | grep websocket

# 检查防火墙
Get-NetFirewallRule | Where-Object { $_.DisplayName -like "*Python*" }
```

#### CANoe/TSMaster连接失败
```powershell
# 检查软件运行状态
Get-Process | Where-Object { $_.ProcessName -match "CANoe|TSMaster" }

# 检查COM接口
Get-ChildItem HKLM:\Software\Classes | Where-Object { $_.Name -match "CANoe|TSMaster" }
```
