## 1. 项目概述

### 1.1 项目简介

**Python Test Executor（测试执行器）** 是一款面向汽车诊断测试的桌面自动化执行平台，支持 CANoe、TSMaster、TTworkbench 三种主流测试工具的统一调度与结果收集。

### 1.2 核心能力

| 能力 | 描述 |
|------|------|
| 多工具统一管理 | 通过适配器模式封装不同测试工具的调用接口 |
| TDM2.0 规范兼容 | 支持 TDM2.0 格式的任务创建、进度跟踪、结果上报 |
| 桌面/Web 双模式 | 支持 PyWebView 桌面应用和 Flask Web 服务两种运行模式 |
| 故障容错 | 断路器模式 + 失败重试（指数退避）确保测试可靠性 |
| 实时监控 | WebSocket 推送任务状态、日志实时显示 |

### 1.3 技术栈

| 层级 | 技术 |
|------|------|
| 桌面框架 | PyWebView 4.x |
| Web 框架 | Flask 2.x + Flask-SocketIO |
| 异步编程 | gevent + monkey patching |
| Windows 集成 | pywin32 (COM 接口) |
| 数据存储 | JSON 文件持久化 |
| 打包工具 | PyInstaller 5.x |

## 3. 入口点分析

```python
# 用途：直接运行 Flask 开发服务器

python main.py
```

**特点：**
- 使用 Flask 内置开发服务器
- 适合调试和快速开发
- **不适用于生产环境**

### 3.2 main_production.py - 生产模式入口

```python
# 用途：PyWebView 桌面应用 + 内嵌 Flask 服务器
python main_production.py
```

**核心功能：**
1. 创建 1200x800 桌面窗口
2. 启动内嵌 Flask 服务器（默认端口 8180）
3. 初始化任务执行器和调度器
4. 注册信号处理器实现优雅关闭
5. 使用 gevent monkey patching 支持异步

**关键组件：**
- `PythonExecutorProduction` - 生产级执行器
- `TaskScheduler` - 任务调度器
- SocketIO WebSocket 通信

兼容性问题保留，现已迁移至 `main_production.py`。

## 5. 核心层模块

### 5.1 task_executor_production.py - 生产级任务执行器

**文件大小：** 60KB+

**核心特性：**

1. **断路器模式（Circuit Breaker）**
   - 失败次数超过阈值时快速失败
   - 防止故障传播

2. **线程池执行**
   - 使用 `ThreadPoolExecutor` 并发执行任务
   - 可配置线程数

3. **任务队列**
   - 内部任务队列管理
   - 优先级调度

**关键方法：**
```python
execute_task()                    # 提交任务到队列
_execute_task_production()        # 实际执行（含断路器）
start()                           # 启动执行器
stop()                            # 停止执行器
cancel_task()                     # 取消任务
retry_task()                      # 重试失败任务
```

### 5.2 task_scheduler.py - 任务调度器

**功能：**
- 延迟任务调度

```python
schedule_task()                   # 调度延迟任务
schedule_periodic_task()         # 调度周期任务
get_scheduled_tasks()            # 列出调度任务
```

## 6. 适配器层架构

### 6.1 适配器模式设计

```
┌─────────────────────────────────────────────────────────┐
│                    BaseTestAdapter                      │
│                    (抽象基类)                            │
├─────────────────────────────────────────────────────────┤
│  tool_type: TestToolType                               │

│  + disconnect() -> bool                                │

│  + stop_test() -> bool                                │
│  + execute_test_item(item) -> Dict                    │
└─────────────────────────────────────────────────────────┘
           ▲                    ▲                    ▲
           │                    │                    │
    ┌──────┴──────┐      ┌──────┴──────┐      ┌──────┴──────┐
    │CANoeAdapter │      │TSMasterAdapter│     │TTworkbenchAdapter│
    └─────────────┘      └──────────────┘      └───────────────┘
```

### 6.2 适配器状态机

```
     ┌─────────┐
     │  IDLE   │◄────────────────────────┐
     └────┬────┘                         │
          │ connect()                    │ disconnect()
          ▼                             │
    ┌─────────────┐                      │
    │ CONNECTING  │                      │
    └──────┬──────┘                      │
           │ success                      │
           ▼                             │
    ┌─────────────┐                      │
    │ CONNECTED   │──────────────────────┤
    └──────┬──────┘                      │
           │ start_test()                │
           ▼                             │
    ┌─────────────┐                      │
    │  RUNNING    │                      │
    └──────┬──────┘                      │
           │ stop_test()                 │
           ▼                             │
    ┌─────────────┐                      │
    │ CONNECTED   │                      │
    └──────┬──────┘                      │
           │ disconnect()                 │
           ▼                             │
    ┌─────────────┐                      │
    │DISCONNECTED │──────────────────────┘
    └─────────────┘

    异常情况: IDLE/CONNECTED/RUNNING ──► ERROR ──► IDLE
```

```python
@dataclass
class Task:

```

```
    │        └──► 保存到 failed_reports.json
    │
    └──► ReportRetryScheduler 调度重试
             │
             ├──► 指数退避: delay = base_delay * (backoff_factor ^ retry_count)
             │         最大延迟: 3600 秒
             │
             ├──► 到达重试时间
             │        │

             │

                      └──► 失败且 retry_count < max_retries
```

## 14. 高级特性

### 14.1 断路器模式

```
正常请求 ──► 断路器 ──► 执行 ──► 成功 ──► 关闭断路器
                     │
                     │ 失败
                     ▼
              失败计数 +1
                     │
                     │ 超过阈值
                     ▼
              打开断路器（快速失败）
                     │
                     │ 冷却时间后
                     ▼
              半开断路器（允许一个请求）
                     │
                     │ 成功 ──► 关闭断路器
                     │ 失败 ──► 打开断路器
```

### 14.2 配置缓存

```
首次请求 ──► 读取配置文件 ──► 缓存到 workspace/cache/configs/
                    │
后续请求 ──► 命中缓存 ──► 直接返回（避免重复 IO）
                    │
                    └──► 缓存过期/清理 ──► 重新读取
```

### 14.3 实时日志

```
测试执行 ──► TaskLogAdapter ──► WebSocket 推送 ──► 前端实时显示
                       │
                       └──► 文件日志 (logs/)
```
