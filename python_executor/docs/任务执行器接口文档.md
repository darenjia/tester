# 任务执行器接口文档

## 一、任务接收接口

### 1. 创建任务接口

**接口地址**: `POST /api/tasks`

**请求头**:
```
Content-Type: application/json
```

**请求体格式 (TDM2.0标准)**:

> 支持两种格式：TDM2.0格式（推荐）和内部格式（兼容）

#### TDM2.0格式示例（真实任务格式）:
```json
{
    "taskNo": "TT26020203",
    "projectNo": "yyy",
    "deviceId": "测试",
    "caseList": [
        {
            "caseName": "Bus Sleep Mode 唤醒_Local",
            "caseNo": "ANM_TG1_TC02_SC01",
            "caseSource": "标准要求",
            "caseType": "",
            "expectedResult": "DUT 发送的所有 CAN 报文类型全部为标准帧格式",
            "maintainer": "夏承璞",
            "moduleLevel1": "通信",
            "moduleLevel2": "",
            "moduleLevel3": "",
            "preCondition": "上电后稳定通信",
            "priority": "P3低",
            "stepDescription": "1) 配置 CANoe 的位定时参数与 DUT 相匹配；2) DUT 上电；..."
        }
    ]
}
```

#### 完整字段说明:
```json
{
    "projectNo": "项目编号",
    "taskNo": "任务编号",
    "taskName": "任务名称",
    "caseList": [
        {
            "moduleLevel1": "一级模块",
            "moduleLevel2": "二级模块",
            "moduleLevel3": "三级模块",
            "caseName": "用例名称",
            "priority": "P3低",
            "caseType": "功能测试",
            "preCondition": "前置条件",
            "stepDescription": "步骤描述",
            "expectedResult": "预期结果",
            "maintainer": "维护人",
            "caseNo": "用例编号",
            "caseSource": "用例来源",
            "changeRecord": "变更记录",
            "tags": "标签"
        }
    ],
    "deviceId": "设备ID",
    "toolType": "canoe|tsmaster|ttworkbench",
    "configPath": "配置文件路径",
    "configName": "配置名称",
    "variables": {},
    "baseConfigDir": "基础配置目录",
    "canoeNamespace": "CANoe命名空间"
}
```

**响应格式**:
```json
{
    "success": true,
    "message": "任务已创建并提交到队列",
    "data": {
        "taskNo": "TT26020203",
        "projectNo": "yyy",
        "deviceId": "测试",
        "caseCount": 1
    }
}
```

## 二、Case 模型字段说明 (TDM2.0标准)

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `moduleLevel1` | string | 是 | 一级模块 |
| `moduleLevel2` | string | 是 | 二级模块 |
| `moduleLevel3` | string | 是 | 三级模块 |
| `caseName` | string | 是 | 用例名称 |
| `priority` | string | 否 | 优先级 |
| `caseType` | string | 否 | 用例类型 |
| `preCondition` | string | 否 | 前置条件 |
| `stepDescription` | string | 否 | 步骤描述 |
| `expectedResult` | string | 否 | 预期结果 |
| `maintainer` | string | 否 | 维护人 |
| `caseNo` | string | 是 | **用例编号（脚本标识）** |
| `caseSource` | string | 否 | 用例来源 |
| `changeRecord` | string | 否 | 用例变更记录 |
| `tags` | string | 否 | 用例标签 |
| `dtcInfo` | string | 否 | DTC信息 |
| `params` | object | 否 | 测试参数 |
| `repeat` | int | 否 | 重复执行次数 |

---

## 三、Task 模型字段说明

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `projectNo` | string | 是 | 项目编号 |
| `taskNo` | string | 是 | 任务编号（唯一标识） |
| `taskName` | string | 是 | 任务名称 |
| `caseList` | array | 是 | 用例列表 |
| `deviceId` | string | 否 | 设备ID |
| `toolType` | string | 否 | 工具类型: canoe/tsmaster/ttman |
| `configPath` | string | 否 | 配置文件路径 |
| `configName` | string | 否 | 配置名称 |
| `variables` | object | 否 | 变量字典 |
| `baseConfigDir` | string | 否 | 基础配置目录 |
| `canoeNamespace` | string | 否 | CANoe命名空间 |
| `timestamp` | string | 否 | 时间戳 |

---

## 四、兼容性说明

1. **TDM2.0兼容**: 系统使用 TDM2.0 标准字段名，支持直接对接 TDM2.0 接口
2. **字段映射**:
   - `caseNo` ↔ 脚本中的用例编号
   - `caseName` ↔ 接口中的用例名称
3. **执行时映射查找**: 任务执行时可根据 `caseNo` 查找对应的用例映射信息

---

## 五、其他相关接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/tasks` | GET | 获取任务列表 |
| `/api/tasks/<taskNo>` | GET | 获取任务详情 |
| `/api/tasks/<taskNo>` | DELETE | 取消任务 |
| `/api/tasks/<taskNo>/result` | GET | 获取任务结果 |
| `/api/case-mappings` | GET | 获取用例映射列表 |
| `/api/case-mappings/sync` | POST | 从任务同步映射 |
| `/api/case-mappings/import` | POST | 批量导入映射 |

## 六、用例映射管理接口

### 6.1 获取映射列表

**接口地址**: `GET /api/case-mappings`

**查询参数**:
- `category`: 分类筛选 (canoe/tsmaster/ttman)
- `enabled`: 启用状态 (true/false)
- `search`: 搜索关键字
- `page`: 页码
- `page_size`: 每页数量

**响应格式**:
```json
{
    "success": true,
    "data": {
        "items": [...],
        "total": 100,
        "page": 1,
        "page_size": 20
    }
}
```

### 6.2 批量导入映射

**接口地址**: `POST /api/case-mappings/import/preview`

**请求体**:
```json
{
    "file_id": "临时文件ID",
    "column_mapping": {
        "Excel列名1": "case_no",
        "Excel列名2": "case_name"
    },
    "default_category": "canoe",
    "sheet_index": 0
}
```

**响应格式**:
```json
{
    "success": true,
    "data": {
        "mappings": [...],
        "total": 100,
        "valid": 98,
        "invalid": 2,
        "errors": [{"row": 5, "errors": ["case_no不能为空"]}]
    }
}
```

### 6.3 执行导入

**接口地址**: `POST /api/case-mappings/import/execute`

**请求体**:
```json
{
    "file_id": "临时文件ID",
    "column_mapping": {...},
    "default_category": "canoe",
    "sheet_index": 0,
    "overwrite": true,
    "changed_by": "web_user",
    "change_reason": "从Excel批量导入"
}
```
